# Практична робота No30. Використання індукції для доведення коректності алгоритму
### Розділ VІІ. Методи доведення та верифікації програмних систем
### Тема 17. Базові методи доведення правильності програм
### Мета: Провести формальне індуктивне доведення коректності обраного алгоритму та підкріпити його ілюстративними тестами.
## Хід роботи
### Крок 1. Оберіть алгоритм і параметр n
Algorithm: bubble_sort(arr)

Parameter n: довжина масиву arr.

Domain: $n \ge 0$.
## Крок 2. Сформулюйте P(n)
$P(n)$: Функція bubble_sort(arr) повертає масив, де всі елементи впорядковані за зростанням (тобто arr[0] <= arr[1] <= ... <= arr[n-1]), для всіх масивів arr довжиною $n$.
## Крок 3. Виконайте доведення (Loop Invariant)
Для доведення коректності алгоритму використовуємо інваріант зовнішнього циклу (де змінна $i$ позначає кількість повністю завершених ітерацій зовнішнього циклу)

Loop invariant: Після $i$-ї ітерації: підмасив arr[n-i .. n-1] є відсортованим за зростанням $\wedge$ ∀x ∈ arr[0 .. n-i-1], ∀y ∈ arr[n-i .. n-1]: x <= y.  

Initialization: Before the first iteration ($i = 0$):   

Поточне значення лічильника зовнішнього циклу $i = 0$ [by code].

Межі підмасиву набувають вигляду arr[n-0 .. n-1], тобто він є порожнім [by algebra].

Порожній підмасив за визначенням є відсортованим за зростанням [by definition].

Умова порівняння з порожньою множиною ∀x ∈ arr[0 .. n-1], ∀y ∈ ∅: x <= y є істинною [by definition]. 

Отже, loop invariant перед першою ітерацією виконується тривіально [by definition].

Maintenance: Assume invariant holds after iteration $i$.

Show it holds after iteration $i+1$.  

Після $i$-ї ітерації підмасив arr[n-i .. n-1] є відсортованим та містить $i$ найбільших елементів масиву [by hypothesis].

На ітерації $i+1$ внутрішній цикл ітерується за індексом $j$ від $0$ до $n-i-2$ [by code].

Тіло циклу порівнює та міняє місцями arr[j] та arr[j+1], якщо виконується умова arr[j] > arr[j+1] [by code].

Завдяки послідовним перестановкам найбільший елемент підмасиву arr[0 .. n-i-1] переміщується на позицію arr[n-i-1] [by definition].

Цей переміщений елемент є меншим або рівним за будь-який елемент із уже відсортованої частини arr[n-i .. n-1] [by hypothesis].

Об'єднання елемента arr[n-i-1] та підмасиву arr[n-i .. n-1] утворює новий відсортований підмасив arr[n-(i+1) .. n-1] [by algebra].

Всі елементи залишеної несортованої частини arr[0 .. n-i-2] $\le$ за елементи з arr[n-(i+1) .. n-1] [by definition].

Отже, loop invariant успішно зберігає істинність для ітерації $i+1$ [by algebra].

Termination: The loop terminates when $i = n$ [by code].

At that point: (підмасив arr[n-i .. n-1] відсортований за зростанням $\wedge$ ∀x ∈ arr[0 .. n-i-1], ∀y ∈ arr[n-i .. n-1]: x <= y) $\wedge$ ($i = n$) $\rightarrow$ підмасив arr[0 .. n-1] повністю відсортований за зростанням [by algebra].  

Оскільки зовнішній цикл завершився при $i = n$, межі підмасиву arr[n-i .. n-1] трансформуються в arr[0 .. n-1] [by algebra].

За припущенням інваріанта, цей підмасив (який тепер охоплює весь масив) є повністю відсортованим за зростанням [by hypothesis].

Отриманий стан структури даних повністю задовольняє математичну властивість результату: $\forall k \in [0..n-2]: res[k] \le res[k+1]$ [by definition].
